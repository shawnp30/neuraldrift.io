## Overview

Custom nodes are the #1 source of ComfyUI problems according to Reddit. This guide covers installation, updates, troubleshooting, and the most essential nodes you should install.

---

## 📦 What Are Custom Nodes?

Custom nodes extend ComfyUI's functionality with new features:
- Advanced samplers
- Video generation
- ControlNet extensions
- UI improvements
- Workflow tools

**Key fact:** Each custom node is an independent plugin, often from different developers, with different update schedules and dependencies.

---

## ✅ How to Install Custom Nodes

### Method 1: ComfyUI Manager (Easiest - Recommended)

**Step 1: Install ComfyUI Manager**

```bash
cd ComfyUI/custom_nodes
git clone https://github.com/ltdrdata/ComfyUI-Manager
```

**Step 2: Restart ComfyUI**

**Step 3: Use the Manager**
- Click "Manager" button in ComfyUI interface
- Search for nodes
- Click "Install"
- Restart when prompted

**Advantages:**
- ✅ One-click install
- ✅ Automatic dependency handling
- ✅ Built-in update system
- ✅ Shows missing nodes in workflows

---

### Method 2: Manual Git Clone

**Step 1: Navigate to custom_nodes folder**

```bash
cd ComfyUI/custom_nodes
```

**Step 2: Clone the node repository**

Example: Installing ComfyUI-Impact-Pack
```bash
git clone https://github.com/ltdrdata/ComfyUI-Impact-Pack
```

**Step 3: Install node dependencies**

```bash
cd ComfyUI-Impact-Pack
pip install -r requirements.txt
```

**Step 4: Restart ComfyUI**

---

### Method 3: Direct Download (Not Recommended)

1. Download node ZIP from GitHub
2. Extract to `ComfyUI/custom_nodes/`
3. Install dependencies manually
4. Restart

**Why not recommended:**
- Hard to update
- Dependency issues common
- No version tracking

---

## 🔄 How to Update Custom Nodes

### Using ComfyUI Manager

1. Open Manager
2. Click "Update All"
3. Restart ComfyUI

---

### Manual Update (Git)

**Update single node:**

```bash
cd ComfyUI/custom_nodes/ComfyUI-Impact-Pack
git pull
pip install -r requirements.txt
```

**Update all nodes at once:**

```bash
cd ComfyUI/custom_nodes
for dir in */; do
  cd "$dir"
  git pull
  cd ..
done
```

**Windows batch script version:**

Create `update_all_nodes.bat`:
```bat
@echo off
cd custom_nodes
for /d %%d in (*) do (
  cd %%d
  git pull
  cd ..
)
pause
```

## 🚨 Common Custom Node Problems

### Problem 1: "Node not found" Error

**Symptoms:**
- Workflow won't load
- Red error message: "Node 'XYZ' not found"

**Cause:**
Missing custom node that the workflow uses

**Solution:**

1. **Identify the missing node:**
   - Error message shows node name
   - Or use ComfyUI Manager → "Install Missing Custom Nodes"

2. **Install the node:**
   - Search in ComfyUI Manager
   - Install exact node pack needed

3. **Restart ComfyUI**

4. **Reload workflow**

---

### Problem 2: Nodes Break After Update

**Symptoms:**
- Workflow worked yesterday
- Now nodes show errors
- Console shows Python errors

**Cause:**
Node update changed behavior or broke compatibility

**Solution:**

**Option A: Rollback the node**

```bash
cd ComfyUI/custom_nodes/PROBLEM_NODE
git log  # Find previous commit hash
git checkout COMMIT_HASH  # Replace with actual hash
```

**Option B: Lock to specific version**

Create `.gitignore` in node folder to prevent updates:
```bash
cd ComfyUI/custom_nodes/PROBLEM_NODE
echo "*" > .git/info/exclude
```

---

### Problem 3: Duplicate Nodes

**Symptoms:**
- Nodes appear twice in menu
- Workflows fail with "duplicate class" errors

**Cause:**
Same node installed multiple times or in different folders

**Solution:**

1. **Find duplicates:**
   ```bash
   cd ComfyUI/custom_nodes
   ls -la
   ```

2. **Delete duplicate folders**
   - Keep only ONE copy of each node

3. **Restart ComfyUI**

---

### Problem 4: Missing Dependencies

**Symptoms:**
- Error: `ModuleNotFoundError: No module named 'XXX'`
- Node appears but doesn't work
- Console shows import errors

**Cause:**
Node's Python dependencies not installed

**Solution:**

```bash
cd ComfyUI/custom_nodes/PROBLEM_NODE
pip install -r requirements.txt
```

**If no requirements.txt exists:**
- Check node's GitHub README
- Install dependencies manually:
  ```bash
  pip install [package-name]
  ```

---

### Problem 5: Node Crashes ComfyUI

**Symptoms:**
- ComfyUI won't start
- Crashes immediately on launch
- Console shows error from specific node

**Solution:**

1. **Temporarily disable the node:**
   ```bash
   cd ComfyUI/custom_nodes
   mv PROBLEM_NODE PROBLEM_NODE.disabled
   ```

2. **Start ComfyUI**

3. **Report issue to node developer** (GitHub Issues)

4. **Wait for fix or find alternative**

## 🧪 Testing New Nodes Safely

### Best Practice: Use a Test Environment

**Option 1: Portable copy**
1. Duplicate your entire ComfyUI portable folder
2. Test new nodes in the copy
3. If stable, add to main installation

**Option 2: Git branch** (Desktop install)
```bash
cd ComfyUI
git checkout -b testing
# Install experimental nodes
# If good: merge to main
# If bad: delete branch
```

```
│   ├── ComfyUI_essentials/

└── experimental/
    └── [testing new nodes here]
```

**Why organize?**
- Easier to find/remove nodes
- Clear separation of stable vs experimental
- Better troubleshooting

## 🔧 Advanced: Node Troubleshooting

### Check Node Logs

```bash
# Start ComfyUI with verbose logging
python main.py --verbose

# Watch for errors from specific node
```

---

### Test Node Isolation

**Disable all nodes except one:**

1. Rename all other node folders to `.disabled`
2. Keep only the suspected problem node
3. Start ComfyUI
4. If starts: problem is node conflict
5. If fails: problem is the node itself

---

### Check Python Environment

```bash
# List installed packages
pip list

# Check for conflicts
pip check

# Verify node dependencies
cd ComfyUI/custom_nodes/PROBLEM_NODE
pip install -r requirements.txt --dry-run
```

## 🚨 Nuclear Option: Fresh Node Install

If everything is broken:

1. **Backup workflow files** (`.json` files)

2. **Delete all custom nodes:**
   ```bash
   cd ComfyUI
   rm -rf custom_nodes/*  # Linux/Mac
   rmdir /s custom_nodes  # Windows
   ```

3. **Reinstall only essential nodes:**
   - ComfyUI-Manager
   - ComfyUI-Impact-Pack
   - Your 3-5 most-used nodes

4. **Test each workflow** before adding more nodes

5. **Add nodes one at a time**, testing after each

## 📝 Node Installation Checklist

Before installing any custom node:

- [ ] Check GitHub stars/forks (popularity indicator)
- [ ] Read recent Issues for known problems
- [ ] Check last update date (abandoned nodes = risky)
- [ ] Verify ComfyUI version compatibility
- [ ] Check if dependencies conflict with existing nodes
- [ ] Backup your working installation first
- [ ] Test on duplicate/test environment

---

## 🆘 When Nodes Won't Work

**Checklist:**

1. [ ] Installed in `ComfyUI/custom_nodes/`?
2. [ ] Dependencies installed (`pip install -r requirements.txt`)?
3. [ ] Restarted ComfyUI?
4. [ ] No duplicate node folders?
5. [ ] ComfyUI version compatible?
6. [ ] Checked node's GitHub Issues?
7. [ ] Tried removing and reinstalling?
